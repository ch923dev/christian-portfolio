01 — Overview

The problem worth solving, and who it was for.

The problem

Agentic coding is trapped in a single chat column. Ideas are linear, context scrolls away, and running more than one agent means juggling tabs and losing the thread. The medium fights the way builders actually think — in branches, in parallel, in space.

Expanse rethinks the environment itself: an infinite canvas where every agent is a node you can place, branch, and connect — so orchestrating a build feels like sketching a system, not typing into a box.

“Thinking is spatial. The tools for building with AI should be too.”

02 — Approach

Key product and engineering decisions.

Agents as spatial nodes

Each agent lives as a card on the canvas with its own context, history, and output. You can fork a node to explore a variation, wire one agent’s output into another’s input, and watch parallel runs unfold side by side instead of one-at-a-time.

Performance on an infinite plane

An infinite, pannable canvas streaming live agent output is the core technical challenge — keeping pan, zoom, and parallel runs fluid is where most of the engineering goes. The result holds up whether you have three nodes or three hundred.

  • Infinite pannable / zoomable canvas workspace
  • Multiple agents running in parallel as independent nodes
  • Branch and connect flows to compose multi-step builds
  • Stays smooth from three nodes to three hundred
03 — How it works

The idea, at a high level.

The core idea: every agent is a node on an infinite canvas, and the canvas itself is both the interface and the data model — node positions, branches, and connections are the program. Save a workspace and you’ve saved an entire reasoning flow, ready to resume or fork.

It’s built to stay fluid from three nodes to three hundred, with agents running in parallel instead of one-at-a-time. Expanse is still in active development, so I’ll let the idea speak here and save the internals for a live walkthrough.
